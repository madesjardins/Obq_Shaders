Obq_Fresnel


Class

Texture [Generator]


Output

Color RGB


Synopsis

Obq_Fresnel is a texture node that outputs colored proportion of light reflected/refracted from a surface based on the refractive indices and the extinction coefficients. Note that this shader is particularly useful for getting the right metal reflection color by driving the ks_color port of a standard shader. A lot of presets are available, but you can also import wavelength dependent complex IORs files (.spd, .csv, .txt) or enter them as strings directly. Creation of an energy conserving 2-layers material can also be done by using the Obq_Fresnel2Standards compound.

Note : Even though this is a newer Obq_Fresnel, older Obq_Fresnel nodes will give the same look. You can press the "Load" preset button to update the look.

Warning : You should not use the Obq_Fresnel for dielectrics (material with refraction), the Use IOR for Fresnel parameter of the standard shader should be used instead. However, you can use the Material Type : Refractive for coatings.


Fig.1 : In this example, the Stanford dragon is shaded using an Obq_Fresnel with an iron preset plugged into the specular color port of the standard shader. Note that the Fresnel parameters of the standard must not be enabled.


Usage

The simplest way is to use Obq_Fresnel is to plug the output in the specular color port of the standard shader like shown above. Using the Obq_Fresnel2standards compounds, you can also achieve multi-layered materials and respect energy conservation.


Fig.2 : Statues of Ajax from left to right : smooth gold, rough copper, rough osmium with silver flakes underneath a smooth acrylic layer, smooth ice and a vegetable oil covered rosewood. The gold and copper Ajax have been created with their respective preset and simply connected to the specular color port of the standard shader. The third and fifth Ajax are using the compounds shown below the render.


Settings


Fresnel Mode [mode]

Different value input mode.

•Custom:enter arbitrary values or load presets from the combo box and button.
•Custom (DEPRICATED Presets):For compatibility with old Obq_Fresnel nodes. Please hit the load button if you want to update this Obq_Fresnel node.
•File:load a file containing wavelengths and their respective refractive indices and extinction coefficients, respectively n [eta] and k. You can load comma-separated values (.csv), tab-separated values (.txt) and pbrt's .spd.

Warning : Hitting the load preset button to update old Obq_Fresnel will change the look.


Input Type [inputType]

There are 2 different custom input types :

•RGB (LEGACY):use RGB values.
•String SPD:use spectral power distribution strings. Essentially the content of the file mode but in preset.

Presets


Presets [presets]

Material presets with their values.

Warning : Presets are now loaded using the load button.

To add a preset :

  • Get a csv or txt file from the web (Filmetrics or Refractive Index) or spd files from pbrt.
  • Select the File mode.
  • Select the proper units for the file.
  • Change the Arnold verbosity level to "warning+info".
  • Render.
  • Open the script editor and look for the line "# INFO : [arnold] IOR RGB Form:".
  • The 2 following lines contains the RGB values for both eta (n) and k.
  • If you want the strings values, simply open the file in a text editor.
  • Edit the Obq_Fresnel.spdl template file. (This is for Softimage, but should be similar in Maya and Houdini)
  • Look for the "preset" parameter declaration and increment the maximum value (remember it too, this is your preset number).
  • Look for the "preset" combo box declaration and add your preset name and preset number in there.
  • Look for the "btnLoadPreset_OnClicked()" function. Add a condition for that new preset number (you can copy+paste a previous preset and change the values).
  • Enter the 2 RGB values and the 3 strings SPD for that preset.

SPD (Strings or File)


Filename [iorFilename]

You can browse for a ior files. While .csv and .txt contains both n and k or only n, spds only contains one of them at a time. Gold, for example, would have a file "Au.eta.spd" for n and a "Au.k.spd" for k. Loading one of them will automatically load the other if it exists.

Note : This parameter is available in file mode.


Lambda [lambdasStr]

A string representing the sampled wavelenghts.

Note : This parameter and the following 2 are available in custom mode with string spd input type.

Note : You can copy the content of a spd file to the proper string parameters. In file mode, when using the verbose "warning+info" mode, the log will show the string values of the loaded file. This is quite practicle if you want to add a new preset.


Eta (n) [etasStr]

A string representing the real part of the complex index of refraction for the sampled wavelenghts.


k [ksStr]

A string representing the imaginary part of the complex index of refraction for the sampled wavelenghts.


Wavelenghts Units [lambdaUnits]

Depending on the origin of the file, wavelenghts units will usually be :

•nanometers:[ 360nm-830nm ].
•micrometers:[ 0.360um-0.830um ].

You have to select the appropriate one.

Warning : A warning will be issued if the shader thinks you selected the wrong units.


RGB Refractive Indices


n (eta) [iorRGB]

RGB n [eta] are the real parts of the complex index of refraction. There is no single specific wavelenght for red, green or blue. It's rather a sum of weighted values. The best way to choose values for them is to actually load the file corresponding to that material, select "warning+info" verbose level in the diagnostic tab of the Arnold render options and look at the log for the actual RGB values.


Extinction Coefficients


k [kRGB]

RGB k (extinction coefficients) are the imaginary parts of the complex index of refraction. Metals absorb and reflect light, that's what gives the color. They are approximately equal to 0 for non metals. Values can be found the same way as for n.


Equations


Materiel Type [method]

The fresnel implementation used for evaluation :

•Refractive:uses equations for dielectrics. Choose this mode if the material reflects and/or refract lights. If you want to simulate a varnished piece of wood, for example, choose this mode and use values of the varnish. Instead of driving the refraction scale or color with the transmitted value, drive the diffuse scale of the shader or premultiply the texture color with it. These equations ignore k.
•Metals:uses equations for metals. Choose this mode if you want to drive the reflection or specular color of a material.

Options


Roughness [roughness]

This approximates the fresnel coefficient for a rough surface using a GTX normal distribution function.

Warning : You should use this parameter with the LUT enable only.


Ratio of polarized Fs/Fp [ratioFsFp]

Ratio of s- and p-polarisations.


Backface [backfaceMode]

This allows you to control what happens when a backface of a polygon is intersected by a ray. This is only used when "Refractive" type is chosen and you want to use on a dielectric even though you're not supposed to ;)

•Same as Frontface:This treats backfaced polygon as frontface.
•Swap IORs:Use this when the refracting material is a closed volume. It swaps the ior and the media in ior. Because it is important to know in which media you are in when calculating these values, being in water and going into glass will not give the same values as being in glass and going into water.
•Always White:Returns white.
•Always Black:Returns black.

Output


Output Refraction Color [transmittance]

By default, this shader returns the reflected portion of light. If this is check, it will return the refracted portion of light. It's essentially equals to 1-R except for metals (0).


Media In


Refractive Indices - n [iorInRGB]

Same as previous n, but this is the media in which your object lies. Generally equals to 1, but if you're in water, for example, it would be 1.33. Ideally, this would exist and would be equal the the ray current IOR.


LookUp Table


Enable precalculated LUT [useLUT]

Precalculate the fresnel term for every sample size ° during the proprocessing phase.

Note : This translate into an acceleration of approximately 7%.


Sample size (°) [LUTSampleSize]

The distance in degrees between 2 samples in the LUT. This will produce a LUT with 90/LUTSampleSize + 1 entries.


Roughness sample size [roughnessSampleSize]

The distance [0,1] between 2 samples in the rough LUT. This will produce a LUT with (90/LUTSampleSize + 1) * (1/roughnessSampleSize +1) entries.


Spectrum


Use full spectrum data in equations (metal) [useFullSpectrum]

Normally, the complex IOR spectral data is transformed into 2 CIE XYZ coordinates and then converted into 2 RGB values representing a complex index of refraction, not a color. These RGB values (eta and k) are then used in the complex fresnel equations. When you enable this option, the fresnel term for every spectral entries is calculated and the result is transformed into a CIE XYZ coordinate and then converted into a RGB value representing a color. This should give very slightly better results.

Note : This option is only available for metals.

Warning : This is longer to process and should always be used with a LUT.


ColorSpace


XYZ to RGB Matrix [xyz2rgb]

This allows you to control the XYZ to RGB matrix in case you need to.


Gamma [outputGamma]

This allows you to control the gamma in case you need to.


Old linear workflow for metal (compatibility) [degamma]

This is the old parameter and it is here for look compatibility between this new and the old Obq_Fresnel.


Color Adjustment


Mode [ccMode]

This allows you to slightly tweak the color (hue and saturation) to match a certain look.

•Off:Don't use any color adjustment.
•HSV:Modify the color using the HSV coordinates system.
•HLS:Modify the color using the HLS coordinates system.

Hue Shift (°) [hueShift]

A standard hue shift in degrees.


Saturation Modifier[saturationMod]

Modifies the saturation using a certain operation.


Saturation Operation[saturationOp]

Uses this operation to affect the saturation by the saturation modifier.
